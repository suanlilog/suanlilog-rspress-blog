# 日常更新流程

记录一下平时写博客、更新文章的标准流程。

---

## 一、写文章

### 1. 新建文件

在 `docs/` 对应的分类目录下新建 `.md` 文件，例如：

```
docs/工具教程/新文章标题.md
```

如果是子分类目录下的文章：

```
docs/工具教程/AI Coding工具/新文章标题.md
```

### 2. 编写内容

直接用 Markdown 语法编写，支持代码块、图片等。图片放在 `docs/public/` 目录下，引用时用 `/图片名.png`。

示例：

```markdown
# 文章标题

这里是正文内容。

​```python
print("hello world")
​```
```

## 二、文章分类

### 现有分类

| 分类 | 目录 | 说明 |
|------|------|------|
| Agent | `docs/Agent/` | 大模型智能体开发 |
| Python教程 | `docs/Python教程/` | Python 工具链与实践 |
| 工具教程 | `docs/工具教程/` | 日常工具配置与使用 |
| 生活 | `docs/生活/` | 生活记录 |

### 新增文章到已有分类

写完文章后，需要更新 `rspress.config.ts` 中的 `sidebar` 配置，让文章出现在侧边栏。

打开 `rspress.config.ts`，找到对应分类的 sidebar 部分，添加一条：

```typescript
{ text: '文章标题', link: '/工具教程/新文章标题' },
```

### 新增子分类

1. 在分类目录下新建文件夹，例如 `docs/工具教程/新子分类/`
2. 在该文件夹下新建文章
3. 在 `rspress.config.ts` 的 sidebar 中添加子分类：

```typescript
{
  text: '新子分类',
  items: [
    { text: '文章一', link: '/工具教程/新子分类/文章一' },
    { text: '文章二', link: '/工具教程/新子分类/文章二' },
  ],
},
```

### 新增顶级分类

1. 在 `docs/` 下新建目录
2. 新建 `index.md` 作为分类首页
3. 在 `rspress.config.ts` 中添加 `nav` 和 `sidebar` 配置

---

## 三、提交流程

### 1. 本地预览

```bash
npm run dev
```

浏览器打开 http://localhost:5173/ 预览效果。

### 2. 提交代码

```bash
git add docs/
git commit -m "feat: 添加新文章《文章标题》"
git push
```

### 3. 等待部署

推送到 GitHub 后，CI/CD 会自动构建并部署。
